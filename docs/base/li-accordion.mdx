# li-accordion 元组件

> li-accordion 是由一组垂直堆叠的带有可点击标题的页面内容片段组成，通过点击标题可以显示或隐藏其相关的页面内容。
> 当在一个页面上显示多个页面部分时，li-accordion 通常用于减少用户滚动页面的操作。

默认情况下，每次只展示一组页面内容。点击打开一组页面内容的时候，其他组的页面内容被折叠起来。
用户也可以控制在同一时刻是否可以同时需要展示多个标题里面的内容。

### 示例

import { LiAccordion } from '@itsi/living-base-react'

### 组件属性

| 属性                     | 类型    | 默认值  | 必选 | 描述                                                                                                           |
| ------------------------ | ------- | ------- | ---- | -------------------------------------------------------------------------------------------------------------- |
| activeSectionName        | string  | 无      | no   | 展开属性指定的页面部分。传入页面折叠部分的标题节点名称。名称大小写敏感。如果，未指定该值，默认打开第一部分内容 |
| allowMultipleSectionOpen | boolean | `false` | no   | 是否允许多个折叠部分同时展开。默认只打开一个折叠区域                                                           |
| title                    | string  | 无      | no   | 当鼠标 hover 在组件时，显示在 tooltip 中的标题                                                                 |
| class                    | string  | 无      | no   | 基于组件自带 CSS 类基础上，自定义的 CSS 类名                                                                   |

### 组件事件

| 事件名称        | 类型   | 描述                                                                                                                                                   |
| --------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| onSectionToggle | action | 当折叠部分被打开是触发。或者当组件第一次加载，至少有一个打开的折叠部分时也会触发。返回值为当前激活的部分名称字符串组成的数组。`openSections：string[]` |

### 插槽

| 插槽名称 | 类型                   | 默认值 | 必选 | 描述                                   |
| -------- | ---------------------- | ------ | ---- | -------------------------------------- |
| body     | `LiAccordionSection[]` | 无     | no   | 折叠的内容部分。折叠内容组件组成的数组 |
