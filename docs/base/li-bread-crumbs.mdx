# li-bread-crumbs 元组件

li-bread-crumbs 面包屑组件为用户可视化展示当前打开资源的路径，并方便用户快速返回路径中各个部分的页面，通常被用于 Header 元素下方。
面包屑组件通常使用水平的有序列表 ol 实现。

### 示例

import { LiBreadCrumbs } from '@itsi/living-base-react'

### 组件属性

| 属性  | 类型            | 默认值 | 必选 | 描述 |
| ----- | --------------- | ------ | ---- | ---- |
| theme | "" \| "default" | ""     | no   | 主题 |

### BreadCrumb 子组件属性

| 属性  | 类型   | 默认值 | 必选 | 描述                                               |
| ----- | ------ | ------ | ---- | -------------------------------------------------- |
| href  | string | ""     | no   | 面包屑条目的链接路径                               |
| label | string | ""     | yes  | 面包屑条目的文本标签                               |
| name  | string | ""     | no   | 面包屑条目子组件的名称，可以用于在回调中标识子组件 |

### 组件事件

| 事件名称 | 描述 | 类型 |
| -------- | ---- | ---- |
| \        | \    | \    |

### 插槽

| 插槽名称 | 描述             |
| -------- | ---------------- |
| default  | 面包屑条目子组件 |

### 使用场景

本节为设计人员和开发人员提供了关于在不同上下文中使用此组件的指导原则

import Callout from '../../src/components/Callout'

<Callout emoji="❤️">
  建议使用场景:
  - 需要在当前页面中显示路径历史的时候
  - 需要为用户提供方面的返回上级页面及以前页面时
  - 当有较深的导航结构时
</Callout>

<Callout type="error" emoji="️🚫">
  避免使用场景：
  不要用面包屑代替heading组件
</Callout>
