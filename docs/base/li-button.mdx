# li-button 元组件

按钮用于界面操作。它们默认具有灰色边框和深色文本的外观。在每个视图上应该只有一个
Primary 样式的按钮，用于页面的主要操作。

### 使用场景

本节为设计人员和开发人员提供了关于在不同上下文中使用此组件的指导原则

import Callout from '@components/global/Callout';

<Callout emoji='❤️'>
  建议使用场景: - 通常情况下，按钮用于用户界面中非连接到其他页面的交互 -
  当需要为用户提供响应用户操作时 -
  用于表单的提交，取消，编辑，保存及打开一个模态框 -
  纯文本的按钮看起来像一个连接，这种按钮通常用于“设置”等场景
</Callout>

<Callout
  type='error'
  emoji='️🚫'>
  避免使用场景： - 避免使用按钮链接到其他页面。而应该使用Link组件 -
  当链接在一个句子中时，避免使用按钮进行链接
</Callout>

### 按钮变体使用说明

| 名称        | 描述                                                                                                                                           |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| default     | 默认样式是最常见的按钮变体。仅当需要调整图元的视觉权重时，才切换到其他变体。请注意，默认样式使用透明背景，因此它可以很好地处理的灯光背景渐变。 |
| primary     | 主样式保留给视图的主操作调用。每个视图只应使用一次，例如，表单末尾的提交操作。此样式也适用于暗背景或图像。                                     |
| secondary   | 辅助样式应用于视图中的辅助操作。当已经有一个主要动作，并且我们想要突出显示另一个具有不同风格的动作时，最常用。                                 |
| negative    | Negative 风格是指暗背景，用于插图顶部。                                                                                                        |
| destructive | Destructive 样式应用于删除数据或使操作难以恢复的操作。                                                                                         |
| plain       | 用于不太重要或不太常见的动作                                                                                                                   |
| disabled    | 用于当前不可用或不再可用的操作                                                                                                                 |
| loading     | 用于指示按下按钮且操作仍在处理时的进度                                                                                                         |
