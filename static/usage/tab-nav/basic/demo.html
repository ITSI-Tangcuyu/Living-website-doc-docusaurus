LDS 简介 Living Design System — 应急行业设计系统 简称 LDS，是一套基于 web component 和现 代化 web 开发标准构建的、专门针对应急行业进行了行业特征功能融合的一整套 WEB 开发 体系和工具集。其核心目标是协助用户解决在创建应急行业大屏、WEB 应用、移动端应用等 多端应用时遇到的各种挑战。 设计原则 在进行设计决策时，我们需要始终牢记这些核心原则，我们鼓励在所有项目的设计和开发过 程中各个团队也采用这些原则。 CSS 框架

LDS 开发体系中的 CSS 框架@itsi/css提供了一系列快捷工具类，可以协助开发人员快速对开发的组件进行样式处理。 @itsi/css 库在实现过程中，通过利用@itsi/tokens 库作为原子支撑，可以让组件的样式，风格，动画高效切换，实现了换肤功能的零代码实现

 命名原则 LDS开发体系使用系统中定义的名称来进行交流，形成一个通俗易懂的语境。 这样在企业中的各种角色的人员比如：项目经理、产品经理、设计师、开发人员、测试人员、部署人员、售后支持等进行沟通交流的时候， 会消除歧义。大家都会充分理解LDS系统中Tokens，组件，CSS工具，文件名，仓库名所包含的各种含义。 因此，命名就必须简洁、有含义并易于朗读。
